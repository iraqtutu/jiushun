# 客户历史服务单自动填写功能 - 设计文档

## 概述

改造新建服务单页面的客户信息自动填写逻辑：当用户输入姓名时，搜索该用户名下所有历史服务单，按机器编号去重后根据结果数量决定是自动填充还是弹出卡片列表供用户选择。

## 现状

当前 `autoFillCustomerInfo` 方法调用 `getCustomerHistory` 云函数，该函数：
- 按客户姓名精确匹配
- 只返回最新一条记录的客户信息（phone、address、distributorName、usageType）
- 不涉及产品信息（机器编号、铭牌照片等）

## 新逻辑

### 前端流程（create.vue）

```
用户输入姓名 → onCustomerNameBlur → autoFillCustomerInfo
                                               │
                          ┌────────────────────┴────────────────────┐
                          ▼                                         ▼
                     单条记录                                    多条记录
                          │                                         │
                          ▼                                         ▼
                  自动填入所有字段                      弹出卡片选择列表
                                                      │
                                       ┌──────────────┴──────────────┐
                                       ▼                             ▼
                                  用户选择                        放弃选择
                                       │                             │
                                       ▼                             ▼
                              填入所选记录所有字段              保留已输入，不自动填
```

### 去重规则

- 按 `product.machineNo` 字段去重
- 同机器编号只保留最新一条记录（按 `create_date` 降序排序）

### 自动填入字段

当结果为单条时，自动填充：

| 区域 | 字段 |
|------|------|
| 客户资料 | 电话、地址、用途、经销商 |
| 产品信息 | 机器编号、发动机号、生产日期、铭牌照片 |

### 卡片列表 UI

**弹窗结构：**
- 标题：选择历史记录
- 列表：每条记录一个卡片，内容：
  - 姓名、电话、地址（第一行）
  - 用途、经销商（第二行）
  - 机器编号、发动机号、生产日期（第三行）
  - 铭牌照片缩略图
- 底部：放弃选择按钮

**卡片布局：**
```
┌─────────────────────────────────────────┐
│ 张三  138****1234                       │
│ 山东省临沂市兰山区xxx路xx号             │
├─────────────────────────────────────────┤
│ 自用  │  某某经销商                      │
├─────────────────────────────────────────┤
│ 机器编号: SN123456  发动机号: EN789012  │
│ 生产日期: 2025-03-15                    │
├─────────────────────────────────────────┤
│ [铭牌照片缩略图]                        │
└─────────────────────────────────────────┘
```

### 放弃选择行为

用户点击"放弃选择"或关闭弹窗时：
- 姓名字段保留用户输入的内容
- 其他字段不自动填入，保持空白
- 弹窗关闭

## 云函数改动

### 新增 action: `searchCustomerHistory`

**请求参数：**
```javascript
{
  customerName: String  // 客户姓名
}
```

**返回数据结构：**
```javascript
{
  code: 0,
  data: [
    {
      name: String,
      phone: String,
      address: String,
      usageType: String,
      distributorName: String,
      machineNo: String,
      engineNo: String,
      productionDate: Number,  // timestamp
      platePhoto: String,       // fileID
      create_date: Number       // 用于判断最新
    }
  ]
}
```

**查询逻辑：**
1. 按 `customer.name` 精确匹配
2. 按 `create_date` 降序排序
3. 返回所有匹配记录
4. 前端按 `machineNo` 去重，每组取第一条

## 边界情况

| 情况 | 处理 |
|------|------|
| 历史记录为空 | 不填写任何字段 |
| 仅一条记录 | 直接自动填入 |
| 多条记录（不同机器） | 弹出卡片列表 |
| 用户放弃选择 | 保留已输入，不自动填 |
| 机器编号为空的历史记录 | 视为独立记录，不去重 |
| 编辑模式 | 不触发自动填写 |
